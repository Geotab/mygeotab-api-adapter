## Upgrading to v5.0.0.1

> **WARNING!** The upgrade instructions provided in this document only apply to MyGeotab API Adapter version **5.0.0**. If upgrading from an earlier version, after downloading the latest application executable package(s), it is necessary to FIRST follow the instructions and any associated guides in the [Change Log](../../CHANGELOG.md) for each successive version up to and including v5.0.0 and THEN follow the instructions below.

This is a **patch release** containing a single bug fix. There are **no database schema changes** and **no configuration file changes** in this release. Upgrading is a simple binary replacement.

### 1 Download

Download the application following the [Download](../../README.md#12-download) instructions in the README.

### 2 Stop Application

Stop the MyGeotab API Adapter if it is running.

### 3 Backup Configuration Files

Backup the existing `appsettings.json` and `nlog.config` files (to keep track of the existing setting values).

### 4 Deploy Application

Replace the existing v5.0.0 application files with the new v5.0.0.1 application files. Restore the existing `appsettings.json` and `nlog.config` files (backed-up in [Step 3](#3-backup-configuration-files)) by copying them over the new ones — they do not need to be modified.

### 5 Start Application

Start the MyGeotab API Adapter application and monitor it and its logs for several minutes to ensure that it is working properly, indicating that the upgrade was successful.
